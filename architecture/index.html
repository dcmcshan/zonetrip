Zone Trip
Simulator Manifesto White paper

System architecture

One microphone, one local model boundary, one derived world model.

Zone Trip separates the real booth from the browser simulation. The real installation is a microphone connected to one Linux box. The public Pages front end is a review surface that simulates the booth, world-model drawer, silence timeout, and optional development text input.

Runtime boundaries

Three deployments, one contract.

The same constitutional processor contract is used across the local booth, Pages simulator, and Cloud Run simulator. Only the local booth is the charter-compliant participant-material runtime.

Real booth

Local Linux box

Microphone capture, Whisper STT, Ollama Gemma, processor service, and durable model.md state run on one machine inside the booth boundary.

Review surface

GitHub Pages

The browser renders the Three.js booth, VAD-based silence cue, world-model drawer, and optional development text input. It is a simulator, not the physical participant interface.

Sizing simulator

Cloud Run GPU

Cloud Run uses an L4 GPU, min-instances=0, and the same in-container Ollama/Gemma path for fidelity testing against equivalent local hardware.

Processing loop

STT feeds the model, but transcript does not survive.

  1. Capture

    Audio enters from the booth microphone or simulator browser.

  2. Transcribe

    Whisper receives temporary audio and returns request-local STT.

  3. Constrain

    The processor reads immutable charter.md and current model.md.

  4. Generate

    The LLM returns derived signals plus a complete replacement model.md.

  5. Persist

    The processor atomically writes only the derived world model.

  6. Discard

    Temporary audio and transcript buffers die with the request.

Durable state

Only model.md survives.

The model file is not source material. It should contain abstracted tensions, contradictions, absences, symbolic patterns, minority signals, open questions, and rejected boundary material.

Allowed

  • Derived signals
  • Uncertainty and contradiction
  • Absences and minority signals
  • Rejected boundary categories

Forbidden

  • Raw transcript or quotes
  • Names, factions, subgroup maps, or identity exposure
  • Rankings, counts, scores, or representational claims
  • Recommendations, mandates, diagnoses, or governance action

Silence and scale-to-zero

No speech is no work.

The browser simulator uses lightweight Web Audio RMS VAD. One minute without detected speech stops capture and dims the eight overhead lights. Cloud Run is deployed with min-instances=0; the visual dimming is a cue that the processor simulator can be idle or scaled to zero.

Development path

Text input is for developers only.

Local installs can enable a drawer text form that posts to /process-stt. It exists to test the STT-output side of the loop without recording audio. The physical booth remains microphone-only.
